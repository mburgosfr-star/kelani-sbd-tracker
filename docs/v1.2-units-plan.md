# Kelani SBD Tracker v1.2 unit support plan

Issue: #2 — Switch between Metric and Imperial units.

## Goal

Allow users to use Kelani in either kilograms or pounds.

Supported units:

- kg
- lb

## Core principle

Kelani should keep storing internal training data in kilograms.

User-facing input and display can use kg or lb depending on the selected preference.

This protects existing user data and keeps program calculations stable.

## User-facing behavior

### Onboarding

New users should be able to choose their preferred weight unit before entering maxes.

Options:

- kg
- lb

If lb is selected, onboarding max inputs show lb.

### Settings

Users should be able to change preferred weight unit later in Settings.

Changing the display unit should not rewrite stored workout data.

### Workouts

Workout weights should display in the selected unit.

If the app stores 100 kg internally:

- kg mode: 100 kg
- lb mode: 220 lb or 220.5 lb depending on rounding rule

### Inputs

When the user enters a weight in lb, Kelani converts it to kg internally.

When the user enters a weight in kg, Kelani stores kg directly.

### Body data

Bodyweight, lean mass, and bone mass should follow the selected weight unit.

Body fat, body water, visceral fat, physique rating, and BMR do not change unit.

### Stats

Stats should display weight values using the selected unit.

Stored data remains unchanged.

### Meet planner

Meet attempts should display and accept the selected unit.

Internally, attempts remain kg.

## Rounding

Initial suggested approach:

- kg mode: round to nearest 2.5 kg where workout loading requires it
- lb mode: display to nearest 5 lb for barbell/workout weights
- bodyweight in lb: allow 0.1 lb display/input
- converted internal values should avoid excessive decimals

This needs testing before implementation.

## Data safety

Existing users should default to kg.

Existing saved data should remain valid.

Export/import should preserve stored kg values and include the unit preference.

## Implementation areas

Likely affected areas:

- translations
- onboarding
- Settings
- program generation
- workout display
- weight editing
- PRs/e1RM
- body data
- stats
- meet planner
- export/import
- backup/autosave

## Not in scope

- stones
- per-lift unit settings
- mixed kg/lb programs
- automatic country detection
- account-based preferences

## Acceptance checklist

- [ ] New user can choose kg or lb during onboarding
- [ ] Existing users default to kg
- [ ] User can change unit in Settings
- [ ] Workout weights display in selected unit
- [ ] Weight inputs convert correctly
- [ ] PRs and e1RMs display correctly
- [ ] Bodyweight displays correctly
- [ ] Meet planner works correctly
- [ ] Export/import preserves data
- [ ] Existing kg data is not broken
